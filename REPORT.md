# Отчет по лабораторной работе

## 1. Что такое Git
Git — это распределенная система контроля версий, которая позволяет отслеживать изменения в файлах, работать над проектом совместно с другими разработчиками и возвращаться к предыдущим версиям кода в случае ошибок. Это стандарт индустрии для управления исходным кодом.

## 2. Список использованных команд
1. `git clone` - клонирование репозитория
2. `git add` - добавление файлов в индекс
3. `git commit` - фиксация изменений
4. `git push` - отправка изменений на удаленный сервер
5. `git switch -c` - создание и переключение на новую ветку
6. `git rm` - удаление файла из репозитория
7. `git config` - настройка пользователя

## 4. Работа с .gitignore
Файл `.gitignore` используется для того, чтобы Git игнорировал служебные файлы, локальные настройки и временные данные. Я добавил туда `.env`, папку `outputs/` и кэш Python, чтобы они не засоряли репозиторий.

## 5. Инцидент с secrets.txt
Использовали команду `git log --oneline -- secrets.txt` для поиска коммита, в котором файл был добавлен.
